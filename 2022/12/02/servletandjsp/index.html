1.Web

就是万维网。也可以理解为网站。我们通过网址访问的一切都是万维网中的资源：

  • 静态资源：指我们访问的时候数据始终不变的称之为静态资源：图片、视频、css、html
  • 动态资源：指我们访问的数据是由程序产生的，不同的时间、不同的设备访问的内容各不相同。Servlet&JSP。

1.1C/S架构和B/S架构

  • c/s : client & server :客户端以及服务端架构：炉石传说、魔兽世界、传奇、奇迹、QQ…
  • b/s：browser & server:浏览器以及服务器架构：开心农场、传奇、梦幻奇缘

1.2Tomcat

  • 保证你的环境变量一定配置了JAVA_HOME

  • 解决启动的时候命令行窗口乱码问题

    • 修改apache-tomcat-9.0.33\conf\logging.properties

    • 默认情况下，这个编码格式UTF-8更改为GBK即可

    • java.util.logging.ConsoleHandler.encoding = GBK

2.Http协议

2.1什么是http协议

超文本传输协议。是一个基于请求和响应的协议。并且是一个无状态的协议。应用层。底层是依赖TCP协议，TCP协议经典三次握手。

2.2HTTP请求报文的格式

  1. 请求首行： 请求方式/地址 URI协议/版本
  2. 请求头： Request Header（请求头有很多，都是key:value）
  3. 空行
  4. 请求正文（请求体）

tip:get请求有头没体。 只有post请求有请求体，post的请求体是用来存放参数

  • 响应报文和请求报文并无什么不一致的

2.3响应状态码

  • 客户端请求成功。
  • 302 : 重定向。
  • 403 ：服务器接收到了请求，但是拒绝提供服务。服务器通常会在响应体中提示为何不提供服务。
  • 404 ：请求的资源不存在。
  • 500 ：服务端发生了不可预期错误，导致无法完成客户端请去。

3.1写servlet其实3种方式

  1. 实现servlet接口
  2. 继承GernericServlet抽象类

3.2请求和响应

3.2.1HttpServletRequest对象

  • 主要方法

    • String getParameter(String name) : 接收请求传递的参数

    • void setCharacterEncoding(String encoding) : 设置每个请求的编码

  • 请求乱码

//1. 接收参数
//1.1 必须要写在接收参数之前
request.setCharacterEncoding("UTF-8");

String username = request.getParameter("username");
String password = request.getParameter("password");

3.2.2HttpServletResponse对象

  • 输出乱码
response.setCharacterEncoding("UTF-8");
//  response.setHeader("Content-type", "text/html;charset=UTF-8");

3.3Servlet配置

3.3.1注解配置

package com.woniu.servlet.web;
import javax.servlet.ServletException;
import javax.servlet.annotation.WebServlet;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import java.io.IOException;

/**
 * name : servlet-name
 * value : url 访问地址
 * loadOnStartup :设置启动速度
 */
@WebServlet(name = "helloServlet", value = "/helloServelt", loadOnStartup = 0)
public class HelloServlet extends HttpServlet {
    @Override
    protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        response.getWriter().write("ok");
    }
}

3.4请求转发和重定向

3.4.1请求转发

地址栏不进行跳转 页面进行跳转

3.4.2重定向

进行两次请求 地址栏URL发生改变

package com.woniu.servlet.web;

import javax.servlet.ServletException;
import javax.servlet.annotation.WebServlet;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import java.io.IOException;

@WebServlet("/forwardServlet")
public class ForwardServlet extends HttpServlet {
    protected void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {

    }

    protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        //1. 接收请求参数：f : 表示进行请求转发， r ：表示进行重定向 （这是我自定义的，不表示真正有具体的操作）
        String method = request.getParameter("name");
        //2. 进行判断
        if ("f".equals(method)) { // 进行请求转发
            // 这个地址可以时一个servlet，也可以是jsp
            request.getRequestDispatcher("/main.jsp").forward(request, response); // 请求转发
        }else if ("r".equals(method)) { // 进行重定向
            response.sendRedirect("/003JavaWeb/helloServelt"); // 重定向
        }else {
            System.out.println("输入有无：请输入：f[请求转发] or r[重定向]");
        }
    }
}

3.4.3请求转发和重定向的区别

  • 请求转发

    • 是服务端行为
    • 只产生一次请求
    • 浏览器地址栏不会改变
    • 请求转发之间，request域中的数据不丢失

  • 重定向

    • 是客户端行为
    • 至少产生两次请求
    • 浏览器地址栏是会发生改变
    • 重定向之间，request域中的数据会丢失，获取数据需要通过session域

4.1过滤器的介绍

package com.woniu.javaweb.web.filter;

import javax.servlet.*;
import javax.servlet.annotation.WebFilter;
import java.io.IOException;

@WebFilter("/*") // 过滤路径:这个配置表示过滤所有的web资源
public class ALoginFilter implements Filter {

    // 临死之前
    public void destroy() {
    }

    // 每次经过过滤路径都会调用一次
    public void doFilter(ServletRequest request, ServletResponse response, FilterChain chain) throws ServletException, IOException {
            chain.doFilter(request, response); // 放行， 当我们的过滤器调用了这个方法就表示，我们放行了过滤条件,就可以直接访问下一个url
    }

    // 出生之后
    public void init(FilterConfig config) throws ServletException {

    }
}

4.2监听器的介绍

  • ServletContextListener : 监听ServletContext对象的创建与销毁
  • HttpSessionListener ： 监听HttpSession的创建与销毁
  • ServeltRequestListener ： 监听ServletRequest的创建与销毁
package com.woniu.javaweb.web.listener;

import javax.servlet.ServletContextEvent;
import javax.servlet.ServletContextListener;
import javax.servlet.annotation.WebListener;

@WebListener
public class MyServletContextListener implements ServletContextListener {

    /**
     * 初始化方法，这个方法会在serveltcontext创建的时候被调用一次
     * @param sce
     */
    @Override
    public void contextInitialized(ServletContextEvent e) {
        System.out.println("------>");
    }

    /**
     * 销毁方法，他会servletcontext被tomcat干掉的时候，调用一次
     * @param sce
     */
    @Override
    public void contextDestroyed(ServletContextEvent e) {
        System.out.println("******>");
    }
}

5.Cookie 和 Session

  • Cookie : 当浏览器访问web服务器的时候，web服务器在响应的时候在HTTP的响应头中附带传送给浏览器的一小段数据。
  • 一旦浏览器保存了cookie，下一次浏览器在访问服务器的时候，就会把cookie再次带回给服务器。
  • cookie保存的数据：key和value
  • cookie是不能跨浏览器和操作系统
  • cookie中保存了一个特殊的字段：JSESSIONID
  • 一旦浏览器设置了不保存cookie，那么session也就失效
